# Auto Sell Automata — репрайсер автозапчастей

Веб-сервис пересчёта цен прайса автозапчастей (б/у детали / разборка / оптовый склад).
Вход — Excel-«проценка» (номенклатура, себестоимость, минимальная рыночная цена);
выход — таблица с рассчитанной «Новой ценой», статусами и готовый xlsx-файл.
Конфликтные позиции (наша цена выше рынка) не решаются автоматически — выносятся
на панель «На разбор».

UI построен по принципу **тайлингового оконного менеджера**: экран — дерево сплитов
из панелей (Прайс / На разбор / Сводка / Настройки правил) на базе Dockview.
Меню «Окна» управляет раскладкой: сплиты вправо/вниз, закрытие, сброс; раскладка
сохраняется между сессиями. Авторизации нет — локальный инструмент.

## Запуск

Нужны: `uv` (Python) и Node.js ≥ 20.

```bash
# терминал 1 — API (порт 8000)
cd backend && uv sync && uv run uvicorn app.main:app --reload --port 8000

# терминал 2 — UI (порт 5173, /api проксируется на 8000)
cd frontend && npm install && npm run dev
```

Или через `make backend` / `make frontend`. Откройте http://localhost:5173,
перетащите xlsx-файл проценки в панель «Прайс».

## Правила расчёта

1. База: `Новая = Min Цена × (1 − скидка)`, дефолт скидки 4%.
2. База ниже себестоимости: есть согласованная уценка → `Уценка × (1 + 5%)`;
   нет → `Себестоимость × (1 + 4%)`.
3. Жёсткий пол: не ниже согласованной уценки.
4. Итог выше Min цены → статус `КОНФЛИКТ`, решение за человеком.
5. Округление: до рубля / копеек / 10 руб.

Все проценты настраиваются в панели «Настройки правил» без перезапуска.

## Структура

- `backend/repricer/` — ядро расчёта (чистые функции, без I/O) + Excel-обвязка (openpyxl);
- `backend/app/` — FastAPI: `POST /api/reprice` (multipart xlsx + параметры → JSON со строками,
  сводкой и base64 результирующего файла);
- `backend/tests/` — pytest: ветки правил, краевые случаи, Excel round-trip, API;
- `frontend/` — Vite + React + TypeScript: Dockview (раскладка), zustand (состояние),
  TanStack Table (таблицы).

## Деплой на Vercel

Репозиторий готов к деплою как есть: `vercel.json` собирает фронтенд в статику
(`frontend/dist`), а `api/index.py` экспортирует то же FastAPI-приложение
serverless-функцией (все `/api/*` переписываются на неё). Достаточно импортировать
репозиторий в Vercel или выполнить `vercel deploy` из корня.

Ограничения serverless: тело запроса/ответа до 4.5 МБ (текущие файлы ~200 КБ —
запас большой), холодный старт Python-функции 1–3 сек после простоя.
